# Open Questions + Future Work

Doc-Class: RFC
Status: draft
Date: 2026-07-07
Owner: Firegrid Architecture
Substrate: idealized

A future version of this RFC may specify:

```txt
standard envelope schema
standard projection schemas
standard claim record semantics
standard completion key formats
standard provider record taxonomy
standard local stdio adapter behavior
standard security labels
standard retention requirements
implementation-family conformance profiles for record names, permission id format, dead-owner choices, adapter reattach policies, and choreography tool schemas
model-to-code trace validation profile
```

This draft intentionally leaves those open to preserve language/runtime/protocol neutrality.
